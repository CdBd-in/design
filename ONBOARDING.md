# CdBd 디자인 볼트 인수인계 문서

> 이 문서는 **옵시디언(Obsidian)을 처음 설치한 새 디자인 팀원**이
> CdBd 디자인 작업 환경을 처음부터 끝까지 셋업하고, 매일 안전하게
> 작업할 수 있도록 안내합니다. 순서대로 따라 하면 됩니다.

---

## 0. 5분 개념 정리 — 이것만 알고 시작하세요

| 용어 | 쉽게 말하면 |
|---|---|
| **옵시디언(Obsidian)** | 메모/문서를 작성하는 앱. 우리는 디자인 기획·가이드를 여기에 씁니다. |
| **볼트(Vault)** | 옵시디언이 여는 "작업 폴더" 하나. 폴더 = 볼트. |
| **깃(Git)** | 파일의 변경 이력을 저장하고 팀원과 동기화하는 시스템. |
| **깃허브(GitHub)** | 그 깃 저장소를 인터넷에 보관하는 곳. 우리 조직은 `CdBd-in`. |
| **저장소(Repository)** | 깃허브에 올라간 프로젝트 하나. 우리는 카테고리마다 1개씩 둡니다. |
| **동기화(Sync)** | 내 변경을 올리고(push) 남의 변경을 받아오는(pull) 것. **자동으로 됩니다.** |

핵심 한 줄: **"카테고리 1개 = 깃허브 저장소 1개 = 옵시디언 볼트 1개"**

---

## 1. 전체 구조 한눈에 보기

CdBd의 모든 디자인 작업은 **카테고리별 독립 저장소**로 나뉘어 있습니다.
그리고 그 위에, 팀 공통 문서를 담는 **허브 저장소** 하나가 있습니다.

```
design/                          ← 허브 저장소 (git) · 공통 문서 보관
├─ ONBOARDING.md                 ← 이 문서
├─ (팀 규칙·공통 가이드 등 공통 문서)
│
├─ cdbd-design-system/           ← 볼트 1 · CdBd 기획 및 디자인 시스템 관리
├─ cdbd-marketing/               ← 볼트 2 · 마케팅 이미지(블로그 등) 제작
├─ cdbd-design-service/          ← 볼트 3 · 디자인 대행 서비스 (고객 자료, 비공개)
├─ cdbd-templates/               ← 볼트 4 · 템플릿 제작
└─ makevu-qrstp/                 ← 볼트 5 · 메이크뷰 QR 스탬프 페이지 디자인
```

- 최상위 `design`는 **그 자체로 깃 저장소(허브)**입니다. 인수인계 문서·팀
  규칙 같은 **공통 문서만** 담습니다. 가볍고 거의 바뀌지 않습니다.
- 그 안의 5개 카테고리 폴더는 **각각 별도의 독립 저장소**입니다. 허브 저장소는
  이 5개 폴더를 `.gitignore`로 무시하므로 서로 간섭하지 않습니다.
- **총 저장소 6개** = 허브 1개 + 카테고리 5개. 한 곳에서 문제가 생겨도 다른 곳에 영향이 없습니다.
- 각 카테고리 폴더는 **독립된 옵시디언 볼트**입니다. (허브 폴더도 볼트로 열어 공통 문서를 봅니다.)
- `cdbd-design-service`는 **고객 대행 작업**이라 비공개(private) 저장소입니다.
- 카테고리는 앞으로 늘어날 수 있고, 그때마다 저장소를 1개 추가하면 됩니다.

> ⚠️ **디자인 시스템 참조 방법:** 다른 카테고리를 작업하다가 디자인 시스템
> 문서를 봐야 하면, 옵시디언에서 `cdbd-design-system` 볼트로 잠깐 전환해
> 확인합니다. (볼트 간 위키링크는 되지 않습니다 — 의도된 설계입니다.)

## 2. 사전 준비물 (셋업 전에 확인)

아래 4가지가 준비돼야 시작할 수 있습니다. 안 되어 있으면 팀 담당자에게 요청하세요.

1. **CdBd 공용 GitHub 계정 로그인 정보** — 팀이 공용으로 사용하는 GitHub
   계정의 아이디/비밀번호. 담당자에게 전달받으세요.
   (이 계정이 이미 `CdBd-in` 조직 멤버라 5개 저장소에 모두 접근됩니다.)
2. **옵시디언(Obsidian) 설치** — https://obsidian.md 에서 다운로드.
3. **GitHub Desktop 설치** — https://desktop.github.com
   (저장소를 처음 내려받고 로그인 인증을 가장 쉽게 처리해 줍니다.)

## 3. 최초 1회 셋업 (순서대로)

### 3-1. 허브 저장소 받기 (= 작업 폴더 생성)

1. GitHub Desktop 실행 → `File > Options > Accounts`에서 **CdBd 공용 GitHub
   계정으로 로그인**. (브라우저가 열리며 로그인 — 토큰을 직접 다룰 필요 없습니다.)
2. `File > Clone repository` → `CdBd-in` 조직에서 **`design`** 저장소를 클론.
   - Local path는 원하는 위치(예: 문서 폴더)로 지정합니다.
   - 클론하면 `design` 폴더가 생깁니다 — **이게 앞으로의 작업 폴더**입니다.

### 3-2. 카테고리 저장소 5개 받기

`File > Clone repository`로 아래 5개를 하나씩 클론합니다.
**Local path는 반드시 방금 받은 `design` 폴더 안**으로 지정하세요.

| 저장소 | 카테고리 |
|---|---|
| `cdbd-design-system` | 기획 및 디자인 시스템 |
| `cdbd-marketing` | 마케팅 이미지 제작 |
| `cdbd-design-service` | 디자인 대행 서비스 |
| `cdbd-templates` | 템플릿 제작 |
| `makevu-qrstp` | 메이크뷰 QR 스탬프 |

> 💡 허브 저장소는 이 5개 폴더를 `.gitignore`로 무시하도록 설정돼 있어,
> 폴더가 안에 들어와도 허브 저장소와 충돌하지 않습니다.
>
> 💡 GitHub Desktop은 **로그인 인증과 최초 복제 용도로만** 씁니다. 인증 정보가
> 컴퓨터에 저장되면 이후 동기화는 옵시디언이 알아서 하므로, 이제 닫아도 됩니다.

### 3-3. 옵시디언에서 볼트 열기

옵시디언을 실행하고, 시작 화면에서 **`Open folder as vault`**를 선택합니다.
**카테고리 5개 + 허브 1개 = 총 6개 폴더**를 각각 한 번씩 볼트로 엽니다.
(허브 볼트 `design`는 이 인수인계 문서와 팀 규칙을 옵시디언에서 보기 위한 것입니다.)
한 번 열어두면 옵시디언이 기억하므로, 이후엔 좌측 하단 볼트 아이콘으로 전환합니다.

### 3-4. 각 볼트에 obsidian-git 플러그인 켜기

플러그인 설정은 **볼트마다 따로** 해야 합니다. 6개 볼트 각각에서:

1. `설정(Settings) > 커뮤니티 플러그인(Community plugins)`으로 이동.
2. **Obsidian Git** 플러그인이 설치/활성화돼 있는지 확인.
   - 없으면 `Browse`에서 "Obsidian Git" 검색 후 설치 → 활성화.
3. Obsidian Git 설정에서 자동 동기화가 켜져 있는지 확인 (보통 기본값으로 충분):
   - `Auto pull interval` — 켜짐 (예: 1~5분)
   - `Auto commit-and-sync interval` — 켜짐 (예: 1~5분)
   - `Pull updates on startup` — 켜짐 권장

이게 끝나면 셋업 완료입니다. 이제 동기화는 **자동**으로 됩니다.

---

## 4. 매일 작업하는 법

1. **옵시디언을 켜고** 작업할 카테고리의 볼트를 선택합니다. (좌측 하단 볼트 전환)
2. **그냥 평소처럼 문서를 쓰고 이미지를 넣습니다.**
3. obsidian-git이 알아서 주기적으로 **남의 변경을 받아오고(pull) 내 변경을 올립니다(push).**
   따로 "저장 버튼"을 누를 필요 없습니다.
4. 작업이 끝나면 옵시디언을 닫기 전에 잠깐 기다려, 마지막 동기화가 끝나게 둡니다.

### 작업 시작 전 습관

> 큰 작업(문서 여러 개 수정, 폴더 이동 등)을 시작하기 전에
> **명령어 팔레트(`Cmd/Ctrl + P`) → "Obsidian Git: Pull"**을 한 번 실행해
> 최신 상태를 받아오세요. 충돌을 크게 줄여줍니다.

## 5. 해도 되는 것 / 하면 안 되는 것

### ✅ 권장

- 한 번에 **한 카테고리(볼트)에 집중**해서 작업하기.
- 새 이미지는 각 볼트의 정해진 첨부 폴더(예: `_attachments/`)에 넣기.
- 큰 작업 전 **Pull 먼저** 하기.
- 문서를 새로 만들거나 크게 바꿨으면, 동기화가 한 번 돌 때까지(1~5분) 기다리기.

### 🚫 피하기

- **같은 파일을 팀원과 동시에 편집하지 않기.**
  슬랙 등으로 "OO 문서 지금 만질게요" 한마디면 충돌이 거의 사라집니다.
- **이미 올라온 이미지를 같은 이름으로 덮어쓰지 않기.**
  이미지는 깃이 병합하지 못해 한쪽이 사라집니다. 수정본은 새 이름으로.
- **`.obsidian` 폴더 안의 파일을 직접 건드리지 않기.** (옵시디언 내부 설정)
- **공용 계정 비밀번호를 문서·저장소·메신저에 평문으로 적지 않기.**
  담당자에게 안전한 방법으로만 전달받으세요.

## 6. 충돌(Conflict)이 났을 때 — 당황하지 마세요

가끔 문서 안에 이런 표시가 갑자기 보일 수 있습니다:

```
<<<<<<< HEAD
내가 쓴 문장
=======
팀원이 쓴 문장
>>>>>>> origin/main
```

이건 **나와 팀원이 같은 곳을 동시에 고쳤다**는 뜻입니다. 깃이 어느 쪽이
맞는지 몰라서 둘 다 보여주는 것입니다. 대처법:

1. **삭제하지 말고 그대로 두기.** 먼저 팀원에게 알리세요.
2. `<<<<<<<`, `=======`, `>>>>>>>` 세 줄을 **지우고**, 위/아래 내용 중
   살릴 것만 남기거나 둘을 합쳐 올바른 최종 문장으로 정리합니다.
3. 저장하면 다음 동기화 때 정상화됩니다.
4. **혼자 판단이 안 서면 절대 임의로 지우지 말고** 팀 담당자에게 문의하세요.
   (잘못 정리하면 한쪽 작업이 사라집니다.)

> 충돌은 "같은 파일 동시 편집"에서만 생깁니다. 5번 규칙(동시 편집 피하기)을
> 지키면 거의 마주칠 일이 없습니다.

## 7. 자주 묻는 질문

**Q. 내가 쓴 게 팀원 화면에 안 보여요.**
A. 동기화 주기(1~5분)를 기다리거나, `Cmd/Ctrl + P → Obsidian Git: Commit-and-sync`를 실행하세요.

**Q. 옵시디언을 껐다 켜니 다른 볼트가 열려요.**
A. 정상입니다. 좌측 하단 볼트 아이콘으로 원하는 카테고리로 전환하세요.

**Q. 디자인 시스템 문서를 마케팅 작업 중에 보고 싶어요.**
A. 볼트를 `cdbd-design-system`으로 잠깐 전환해 확인하고, 다시 돌아오세요.
   (옵시디언 창을 두 개 띄워 각각 다른 볼트를 열어둘 수도 있습니다.)

**Q. 새 카테고리가 생겼어요.**
A. 담당자가 새 저장소를 만들면, 3-2 ~ 3-4 단계를 그 저장소에 한 번 더 반복하면 됩니다.
