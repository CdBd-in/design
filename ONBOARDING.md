# CdBd 디자인 볼트 인수인계 문서

> 이 문서는 **옵시디언(Obsidian)을 처음 설치한 새 디자인 팀원**이
> CdBd 디자인 작업 환경을 처음부터 끝까지 셋업하고, 매일 안전하게
> 작업할 수 있도록 안내합니다. 순서대로 따라 하면 됩니다.

---

## 0. 5분 개념 정리 — 이것만 알고 시작하세요

| 용어 | 쉽게 말하면 |
|---|---|
| **옵시디언(Obsidian)** | 메모/문서를 작성하는 앱. 우리는 디자인 기획·가이드를 여기에 씁니다. |
| **볼트(Vault)** | 옵시디언이 여는 "작업 폴더" 하나. 폴더 = 볼트. |
| **깃(Git)** | 파일의 변경 이력을 저장하고 팀원과 동기화하는 시스템. |
| **깃허브(GitHub)** | 그 깃 저장소를 인터넷에 보관하는 곳. 우리 팀은 `CdBd-in` 계정을 공용으로 씁니다. |
| **저장소(Repository)** | 깃허브에 올라간 프로젝트 하나. 우리는 카테고리마다 1개씩 둡니다. |
| **SSH 키** | 깃허브에 "이 사람이 맞다"고 알려주는 디지털 신분증. 한 번 만들면 비밀번호를 다시 칠 일이 없습니다. |
| **동기화(Sync)** | 내 변경을 올리고(push) 남의 변경을 받아오는(pull) 것. **자동으로 됩니다.** |

핵심 한 줄: **"카테고리 1개 = 깃허브 저장소 1개 = 옵시디언 볼트 1개"**

## 1. 전체 구조 한눈에 보기

CdBd의 모든 디자인 작업은 **카테고리별 독립 저장소**로 나뉘어 있습니다.
그 위에, 팀 공통 문서를 담는 **허브 저장소** 하나가 있습니다.

```
design/                          ← 허브 저장소 · 공통 문서 보관
├─ ONBOARDING.md                 ← 이 문서
├─ (팀 규칙·공통 가이드 등 공통 문서)
│
├─ cdbd-design-system/           ← 볼트 1 · CdBd 기획 및 디자인 시스템 관리
├─ cdbd-marketing/               ← 볼트 2 · 마케팅 이미지(블로그 등) 제작
├─ cdbd-design-service/          ← 볼트 3 · 디자인 대행 서비스 (고객 자료, 비공개)
├─ cdbd-templates/               ← 볼트 4 · 템플릿 제작
└─ makevu-qrstp/                 ← 볼트 5 · 메이크뷰 QR 스탬프 페이지 디자인
```

- 최상위 `design`은 **그 자체로 깃 저장소(허브)**입니다. 인수인계 문서·팀
  규칙 같은 **공통 문서만** 담습니다.
- 그 안의 5개 카테고리 폴더는 **각각 별도의 독립 저장소**입니다. 허브는
  이 5개 폴더를 `.gitignore`로 무시하므로 서로 간섭하지 않습니다.
- **총 저장소 6개** = 허브 1개 + 카테고리 5개. 한 곳에서 문제가 생겨도 다른 곳에 영향이 없습니다.
- 각 카테고리 폴더는 **독립된 옵시디언 볼트**입니다.
- `cdbd-design-service`는 **고객 대행 작업**이라 비공개(private) 저장소입니다.

> ⚠️ **디자인 시스템 참조 방법:** 다른 카테고리를 작업하다가 디자인 시스템
> 문서를 봐야 하면, 옵시디언에서 `cdbd-design-system` 볼트로 잠깐 전환해
> 확인합니다. (볼트 간 위키링크는 되지 않습니다 — 의도된 설계입니다.)

## 3. 최초 1회 셋업

### 3-1. SSH 키 만들기 (깃허브용 디지털 신분증)

내 컴퓨터와 깃허브가 서로를 안전하게 알아보도록 키 한 쌍을 만듭니다.
**기기마다 별도로** 만들어야 합니다 (회사 컴퓨터·노트북 두 대를 쓰신다면 각각).

**Step 1.** 터미널에서 기존 키가 있는지 확인:

```bash
ls -la ~/.ssh/
```

- 폴더가 없거나 `id_ed25519` 같은 파일이 보이지 않으면 → 다음 Step으로
- 이미 `id_ed25519` 또는 `id_rsa` 파일이 있으면 → **담당자에게 문의** (덮어쓰면 다른 서비스 인증이 망가질 수 있음)

**Step 2.** SSH 키 생성. 이메일 자리에는 본인 식별용 라벨을 넣습니다 (예: `MacBook-홍길동-2026-05-21`):

```bash
ssh-keygen -t ed25519 -C "MacBook-홍길동-2026-05-21" -f ~/.ssh/id_ed25519
```

**Step 3.** 패스프레이즈를 묻습니다 (2번):

```
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
```

> 💡 **패스프레이즈는 꼭 설정하세요.** 짧고 복잡한 것보다 **여러 단어를 띄어서 조합**한 문장이 더 안전합니다.
> 예: `푸른하늘 노란새 빨간자전거 2026`. macOS 키체인이 기억해 주므로 평소엔 안 칩니다.
> **이 패스프레이즈를 1Password 같은 곳에 안전하게 저장**해 두세요. 잊으면 키를 새로 만들어야 합니다.

**Step 4.** 키가 macOS에 자동으로 등록되게 설정:

```bash
eval "$(ssh-agent -s)"
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

두 번째 명령에서 방금 만든 패스프레이즈를 묻습니다 → 입력하면 키체인에 저장됩니다.

**Step 5.** 컴퓨터를 껐다 켜도 자동 동작하게 설정:

```bash
cat > ~/.ssh/config << 'EOF'
Host github.com
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519
EOF

chmod 600 ~/.ssh/config
```

### 3-2. 깃허브에 공개키 등록

**Step 1.** 공개키 내용을 클립보드에 복사:

```bash
pbcopy < ~/.ssh/id_ed25519.pub
```

> 💡 공개키(`.pub`)는 노출돼도 안전한 정보입니다. 개인키(`.pub`이 안 붙은 파일)는 절대 누구에게도 보내지 마세요.

**Step 2.** 브라우저에서 **CdBd 공용 계정**으로 깃허브에 로그인 후, 다음 주소 접속:

```
https://github.com/settings/keys
```

**Step 3.** 우측 `New SSH key` 클릭 후 입력:
- **Title**: `MacBook-홍길동` 같이 본인 기기 식별용 라벨
- **Key type**: `Authentication Key` (기본값)
- **Key**: `⌘+V`로 Step 1에서 복사한 키 붙여넣기

**Step 4.** `Add SSH key` 클릭 → 비밀번호 또는 2FA 확인.

**Step 5.** 등록이 잘 됐는지 터미널에서 확인:

```bash
ssh -T git@github.com
```

처음 실행 시 `Are you sure you want to continue connecting?` → **`yes`** 입력.

다음 메시지가 보이면 성공:
```
Hi CdBd-in! You've successfully authenticated, but GitHub does not provide shell access.
```

### 3-3. 작업 폴더 만들고 저장소 6개 받기

**Step 1.** 작업 폴더가 들어갈 위치로 이동 (예: 사용자 폴더 안 `Documents/GitHub`):

```bash
mkdir -p ~/Documents/GitHub
cd ~/Documents/GitHub
```

**Step 2.** 허브 저장소부터 받기:

```bash
git clone git@github.com:CdBd-in/design.git
cd design
```

**Step 3.** 허브 안에 카테고리 저장소 5개 받기:

```bash
git clone git@github.com:CdBd-in/cdbd-design-system.git
git clone git@github.com:CdBd-in/cdbd-marketing.git
git clone git@github.com:CdBd-in/cdbd-design-service.git
git clone git@github.com:CdBd-in/cdbd-templates.git
git clone git@github.com:CdBd-in/makevu-qrstp.git
```

**Step 4.** 결과 확인:

```bash
ls
```

다음 폴더들이 보여야 합니다: `cdbd-design-system`, `cdbd-marketing`, `cdbd-design-service`, `cdbd-templates`, `makevu-qrstp` + 허브 파일(`ONBOARDING.md`, `README.md` 등).

> 💡 비밀번호도 토큰도 묻지 않고 6번 모두 통과되면 정상입니다.
> SSH 키 덕분에 자동 인증되고 있는 것입니다.

### 3-4. 옵시디언에서 볼트 6개 열기

옵시디언을 실행하고, 시작 화면 또는 좌측 하단 볼트 아이콘에서 **`Open folder as vault`**를 선택합니다.

**아래 6개 폴더를 각각 한 번씩** 볼트로 엽니다 (카테고리 5개 먼저, 허브를 마지막에):

1. `~/Documents/GitHub/design/cdbd-design-system`
2. `~/Documents/GitHub/design/cdbd-marketing`
3. `~/Documents/GitHub/design/cdbd-design-service`
4. `~/Documents/GitHub/design/cdbd-templates`
5. `~/Documents/GitHub/design/makevu-qrstp`
6. `~/Documents/GitHub/design` (허브, 마지막에)

> 첫 오픈 시 "Trust author and enable plugins?" 같은 다이얼로그가 뜨면 **`Trust author and enable plugins`** 선택.

한 번 열어두면 옵시디언이 기억하므로, 이후엔 좌측 하단 볼트 아이콘으로 전환합니다.

### 3-5. 각 볼트에 obsidian-git 플러그인 설치

플러그인 설정은 **볼트마다 따로** 해야 합니다. 6개 볼트 각각에서 진행합니다.

> 💡 **이미 설치돼 있는 볼트는 건너뛰세요.**
> 일부 볼트(특히 `cdbd-design-system`)는 이전 작업자가 셋업한 상태 그대로 받아오기 때문에 obsidian-git이 **이미 설치·활성화돼 있을 수 있습니다.** 좌측 사이드바에 `Obsidian Git` 항목이 보이면 설치된 것입니다. 이 경우 아래 1~4단계(설치)는 건너뛰고, **5단계(설정 확인)와 6단계(검증)만** 진행하세요.

1. `설정(⌘+,) > 커뮤니티 플러그인(Community plugins)` 이동.
2. 처음 여는 볼트라면 **`커뮤니티 플러그인 켜기`** 버튼으로 활성화 (안전 경고 동의).
3. **`찾아보기(Browse)`** → 검색창에 **`obsidian git`** 입력.
4. **`Obsidian Git`** (작성자: **Vinzent03**) 설치 → 활성화.
5. 좌측 사이드바에 추가된 `Obsidian Git` 설정으로 이동, 자동 동기화 옵션이 다음과 같이 설정돼 있는지 확인 (다르면 아래 값으로 맞춤):
   - **Auto pull interval (minutes)**: `10`
   - **Auto commit-and-sync interval (minutes)**: `10`
   - **Pull on startup**: ON
6. 검증: `⌘+P` → `Obsidian Git: Commit-and-sync` 실행 → `No changes to commit` 메시지가 뜨면 OK.

이게 끝나면 셋업 완료입니다. 이제 동기화는 **자동**으로 됩니다.

## 4. 매일 작업하는 법

1. **옵시디언을 켜고** 작업할 카테고리의 볼트를 선택합니다 (좌측 하단 볼트 전환).
2. **그냥 평소처럼 문서를 쓰고 이미지를 넣습니다.**
3. obsidian-git이 알아서 주기적으로 **남의 변경을 받아오고(pull) 내 변경을 올립니다(push).**
   따로 "저장 버튼"을 누를 필요 없습니다.
4. 작업이 끝나면 옵시디언을 닫기 전 잠깐 기다려, 마지막 동기화가 끝나게 둡니다.

### 작업 시작 전 습관

> 큰 작업(문서 여러 개 수정, 폴더 이동 등)을 시작하기 전에
> **명령어 팔레트(`⌘+P`) → "Obsidian Git: Pull"**을 한 번 실행해
> 최신 상태를 받아오세요. 충돌을 크게 줄여줍니다.

## 5. 해도 되는 것 / 하면 안 되는 것

### ✅ 권장

- 한 번에 **한 카테고리(볼트)에 집중**해서 작업하기.
- 새 이미지는 각 볼트의 정해진 첨부 폴더(예: `_attachments/`)에 넣기.
- 큰 작업 전 **Pull 먼저** 하기.
- 문서를 새로 만들거나 크게 바꿨으면, 동기화가 한 번 돌 때까지(약 10분) 기다리기.

### 🚫 피하기

- **같은 파일을 팀원과 동시에 편집하지 않기.**
  슬랙 등으로 "OO 문서 지금 만질게요" 한마디면 충돌이 거의 사라집니다.
- **이미 올라온 이미지를 같은 이름으로 덮어쓰지 않기.**
  이미지는 깃이 병합하지 못해 한쪽이 사라집니다. 수정본은 새 이름으로.
- **`.obsidian` 폴더 안의 파일을 직접 건드리지 않기** (옵시디언 내부 설정).
- **공용 계정 비밀번호·SSH 개인키를 문서·저장소·메신저에 평문으로 적지 않기.**
- **다른 사람의 SSH 개인키 파일(`id_ed25519`)을 복사해서 쓰지 않기.**
  키는 사람마다 자기 것을 만들어야 합니다.

## 6. 충돌(Conflict)이 났을 때 — 당황하지 마세요

가끔 문서 안에 이런 표시가 갑자기 보일 수 있습니다:

```
<<<<<<< HEAD
내가 쓴 문장
=======
팀원이 쓴 문장
>>>>>>> origin/main
```

이건 **나와 팀원이 같은 곳을 동시에 고쳤다**는 뜻입니다. 깃이 어느 쪽이
맞는지 몰라서 둘 다 보여주는 것입니다. 대처법:

1. **삭제하지 말고 그대로 두기.** 먼저 팀원에게 알리세요.
2. `<<<<<<<`, `=======`, `>>>>>>>` 세 줄을 **지우고**, 위/아래 내용 중
   살릴 것만 남기거나 둘을 합쳐 올바른 최종 문장으로 정리합니다.
3. 저장하면 다음 동기화 때 정상화됩니다.
4. **혼자 판단이 안 서면 절대 임의로 지우지 말고** 팀 담당자에게 문의하세요.
   (잘못 정리하면 한쪽 작업이 사라집니다.)

> 충돌은 "같은 파일 동시 편집"에서만 생깁니다. 5번 규칙(동시 편집 피하기)을
> 지키면 거의 마주칠 일이 없습니다.

## 7. 자주 묻는 질문

**Q. 내가 쓴 게 팀원 화면에 안 보여요.**
A. 동기화 주기(약 10분)를 기다리거나, `⌘+P → Obsidian Git: Commit-and-sync`를 실행하세요.

**Q. 옵시디언을 껐다 켜니 다른 볼트가 열려요.**
A. 정상입니다. 좌측 하단 볼트 아이콘으로 원하는 카테고리로 전환하세요.

**Q. 디자인 시스템 문서를 마케팅 작업 중에 보고 싶어요.**
A. 볼트를 `cdbd-design-system`으로 잠깐 전환해 확인하고, 다시 돌아오세요.
   (옵시디언 창을 두 개 띄워 각각 다른 볼트를 열어둘 수도 있습니다.)

**Q. 새 카테고리가 생겼어요.**
A. 담당자가 새 저장소를 만들면, 3-3 ~ 3-5 단계를 그 저장소에 한 번 더 반복하면 됩니다.

**Q. 노트북을 한 대 더 쓰는데 거기서도 작업하고 싶어요.**
A. 3-1(SSH 키 만들기)을 그 기기에서도 **별도로** 진행하세요. 같은 깃허브 계정에 키를 여러 개 등록할 수 있습니다.
   기기마다 자기 키를 갖는 것이 안전한 방식입니다.

**Q. `Permission denied (publickey)` 에러가 떠요.**
A. SSH 키가 깃허브에 등록 안 됐거나 키체인에 안 잡혀 있다는 뜻입니다. 3-2 Step 5의 `ssh -T git@github.com` 명령으로 인증이 되는지 먼저 확인하고, 안 되면 담당자에게 문의하세요.

**Q. 패스프레이즈를 잊었어요.**
A. 옛 키는 폐기하고 새로 만들어야 합니다. 깃허브에서 옛 공개키를 삭제(`https://github.com/settings/keys` → 해당 키 Delete)하고, 3-1부터 다시 진행하세요.
